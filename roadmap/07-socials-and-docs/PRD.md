# 07 — Socials & Docs — PRD

## Problem

Ralphy's outward surface — landing, README, Mintlify docs, brand assets, social posts — is partially built and inconsistent:

1. **Landing makes promises the CLI doesn't yet deliver verbatim.** [`landing/components/Hero.tsx`](../../landing/components/Hero.tsx) shows `ralphy trend`, `ralphy clone`, `ralphy make`, `ralphy iterate`, `ralphy mcp`, `ralphy skill install` — six verbs, none of which exist in `cli/commands/` today (cross-link [`01.01`](../01-cli/SPEC.md)). The fix is on the CLI side; this category owns alignment.
2. **Mintlify exists but is incomplete.** `docs-mintlify/` has `quickstart.mdx`, `introduction.mdx`, `concepts.mdx`, and stubs under `cli/` and `reference/`. The CLI reference isn't auto-generated; it drifts on every verb change. The "Cookbook" section that competitor docs (Stripe, Vercel, Anthropic) make central — doesn't exist for us.
3. **README is the only doc most users will read first.** It's solid but missing: a 60-second video demo, a clear "5 things to try first", and a link to the template gallery.
4. **No template gallery on the landing.** Cross-link [`02.07`](../02-prompts-and-templates/) — users want to see the styles, click "try this".
5. **Brand identity is partial.** Logo exists (`docs-mintlify/logo.png`), favicon exists, README banner exists, but no design system, no social-post template, no consistent palette/typography reference across the three surfaces (landing, docs, README).
6. **Social presence is zero.** No Twitter/X account, no GitHub Discussions, no Discord. For an OSS launch, at least one "where the conversation happens" venue is the bar.

This category owns everything a user sees *before they install* and the documentation they read *after they install*.

## Users

| User | Need |
|---|---|
| **Prospective user (landing)** | Sees the promise, sees a demo, understands what it does in 30 seconds, finds the install command. |
| **New user (docs)** | 30-minute path from "I just installed" to "I shipped a video". |
| **Returning user (reference)** | CLI reference for the verb they forgot the flags for; matches `--help` exactly. |
| **Contributor (CONTRIBUTING.md)** | Knows how to add a playbook, a template, a recipe. |
| **OSS audience (community)** | A place to ask questions, share templates, report bugs. |

## User stories

1. As a **prospective user**, I land on ralphy.dev, see a 60s demo video, see the 4 headline claims (trend / clone / make / iterate), see the install line, copy-paste, install.
2. As a **new user**, I follow Mintlify quickstart: install → setup → make → render → eval. Five pages, each ≤ 5 min reading.
3. As a **returning user**, I search "ralphy generate video flags", land on the autogenerated reference page, find the per-model whitelist, run my command.
4. As a **template-curious user**, I click "Templates" in the landing nav, see a grid of 42 templates with thumbnails, click one, see composition + sample render + "try it" snippet.
5. As an **OSS prospect**, I find Ralphy via Hacker News / Twitter, click the GitHub badge, see the README with screenshots, follow it to docs.
6. As a **contributor**, I read CONTRIBUTING.md, run `ralphy skill new <name>` (per [`03.06.01`](../03-skills/SPEC.md)) and ship a PR.
7. As a **bug reporter**, I find the link to GitHub Issues from the docs, file a structured report using the issue template.

## Success metrics

| Metric | Target at v1.0 | How we measure |
|---|---|---|
| Landing claims with matching CLI verb + `--help` example | 4/4 headline claims | Grep test in CI (already targeted in `01.03.02`) |
| Mintlify quickstart "install → first render" time | ≤ 15 min by stopwatch on a clean machine | Manual |
| CLI reference pages auto-generated from `cli/commands/` | 100% | CI builds the docs |
| Time to find any flag on docs site | ≤ 30s via search | Manual |
| Template gallery live and clickable | Yes | Manual |
| README has demo video + 5-things-to-try + install | Yes | Manual |
| Brand consistency across landing + docs + README | Same logo, palette, typography across all three | Visual audit |
| OSS community channel | At least 1 (GitHub Discussions OR Discord) | Manual |
| 0 broken links in docs | Yes | `lint:docs-links` CI check |

## Non-goals

- **A blog with weekly posts.** Out of scope for v1.0. README + docs + landing is the writeable surface.
- **Translations.** English only.
- **Paid hosting / SaaS marketing.** OSS launch only.
- **Press / launch playbook.** Out of scope; the landing + README are good enough for organic.
- **A community Discord with moderators / mod policy.** GitHub Discussions is the v1.0 channel.
- **Auto-screenshot generation for landing.** Stays manual.
