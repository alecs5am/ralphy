# Templates — category map

54 repo-public templates organized by **segment × persona** — who's *watching* the content and why. Pick the category that matches the audience archetype, then drill into the slug.

> Slug → category is **physical** on disk (`templates/<category>/<slug>/`). The `ralphy template list / show / use / suggest` verbs resolve by slug regardless of category, so you can keep calling `ralphy template use before-after-product --project <id>` as before. Category folders are navigation aid only.

For the cross-category Top-20 viral-2026 playlist, see [`TOP.md`](TOP.md).

## Adding a new template

1. **Pick a category** by audience-persona (not by visual style). If a template fits two, pick the dominant *buyer* / *watcher* — the one who'll convert / share.
2. Create `templates/<category>/<slug>/` with `template.json`, `TEMPLATE.md`, and the format-specific files (`hooks.md` + `prompt-cookbook.md` for vibe-style; `composition.md` + `model-stack.md` + `reference-example.md` + `fragments.md` for vibe-reference).
3. Register: `ralphy template register <slug>`.
4. Verify: `ralphy template suggest "<canonical utterance>"` returns it with `score ≥ 0.5`.
5. Update this file and [`TOP.md`](TOP.md) if it belongs in the Top-20.
