<Note>
  Templates ship in **two places**: `templates/` in the repo (committed, organized into 5 segment-persona categories) and `workspace/templates/` (user-local, flat). Workspace overrides repo on id collision. Both surfaces are read by `ralphy template list / suggest / show / use`.
</Note>

## Quick start

```bash
ralphy template suggest "modern saas product video"     # match a template by utterance
ralphy template list -p                                  # browse all (pretty)
ralphy template use ugc-selfie-product-review --project espresso-001
```

## Author your own

Author a new template by dropping a `templates/<category>/<slug>/` folder with:

- `template.yaml` — id, kind, requires, suggest keywords, frontmatter (`version: 1`)
- `composition.md` — the production blueprint (scene list, camera grammar, prompt patterns)
- *(optional)* `reference.mp4` — a rendered Ralphy output that proves the template works

Validate with:

```bash
ralphy template show <slug>
ralphy template register <slug>
```

See the [authoring templates guide](/authoring/templates) for the full schema and worked-example pattern.

## Mint a template from a public URL

`ralphy clone <url>` chains `ref pull` → `ref analyze` → `ref blueprint` → writes a new vibe-style template under `workspace/templates/clone-<slug>/`. Edit the generated `composition.md`, then register it.

```bash
ralphy clone https://tiktok.com/@x/video/72939...
```
