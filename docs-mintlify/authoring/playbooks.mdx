<Note>
  Ralphy doesn't ship a single "do everything" agent. It ships **role-specific playbooks** the agent reads on demand based on the user's intent. The router lives at [`AGENTS.md`](https://github.com/alecs5am/ralphy/blob/main/AGENTS.md). Every playbook is markdown in `docs/playbooks/`, source-of-truth in the repo.
</Note>

## Composition

A request that spans roles is a chain in role order. Example: "make a video in the style of <url> for <brand>" → researcher → scenarist → art-director → editor. The producer playbook is the wrapper for end-to-end.

## Hard invariants (every playbook honors these)

1. **No FAL_KEY, no Vercel, no OpenAI direct.** Only `OPENROUTER_API_KEY` + `ELEVENLABS_API_KEY`.
2. **`ralphy` is the only entry point** for model calls, ffmpeg recipes, yt-dlp pulls, and project mutations.
3. **Reference-required gate.** Named person, brand, or specific real entity → require user-supplied reference before any generation.
4. **Quality gates refuse, not warn.** Two consecutive failures → stop and report concrete options.
5. **Always check `MODELS.md`** before any model call — Claude's training is stale on model versions.
6. **Append-only on generations.** Regen → new version (`.v2.<ext>`), never overwrite.

Full invariant list: [`AGENTS.md` § Hard invariants](https://github.com/alecs5am/ralphy/blob/main/AGENTS.md#hard-invariants-apply-across-all-playbooks).

## Author your own playbook

Add a markdown file under `docs/playbooks/<role>.md` and a row to the routing table in `AGENTS.md`. Optionally wrap it as a slash-command skill under `.agents/skills/<role>/SKILL.md` so users can invoke it directly. See the [skill authoring guide](/authoring/skills).
